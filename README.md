# Synera: Synergy Auto-Arena

一款基于 **Qt 6.8 + MSVC 2022** 的桌面自走棋（Auto-Battler）游戏。

> 灵感来源于《The Last Flame》和《Teamfight Tactics》。

---

## 快速开始（Release）

从 [GitHub Releases](https://github.com/ZijinWukll/AutoChessDemo/releases) 下载最新版本的 `Synera-v4.0.zip` 压缩包。

1. 将下载的 ZIP 压缩包**完整解压**到一个文件夹（不要直接拖出 exe）
2. 进入文件夹，双击 **SYNERA.exe** 即可开玩
3. **推荐**：右键 SYNERA.exe → 发送到 → 桌面快捷方式，方便以后快速启动

> **注意**：SYNERA.exe 依赖同目录下的 Qt DLL 文件才能运行，请保持解压后的所有文件在同一个文件夹内，不要单独移动 exe 文件。如果缺少 VC++ DLL，请安装 [VC++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)。

## 快速开始（Developer）

### 环境要求

| 工具          | 版本                                    |
| ------------- | --------------------------------------- |
| Visual Studio | 2022（含"使用 C++ 的桌面开发"工作负载） |
| Qt            | 6.8.3 (MSVC 2022 64-bit)                |
| Windows SDK   | 10.0                                    |

### 构建 & 运行

1. **安装 Qt 6.8.3**，选择 MSVC 2022 64-bit 组件
2. **配置 Qt MSBuild**：Visual Studio 中打开菜单 `扩展 → Qt VS Tools → Qt Versions`，添加 `D:\Qt\6.8.3\msvc2022_64`
3. **打开解决方案**：双击 `AutoChessDemo.slnx`
4. **按 F5** 构建并运行

### 命令行构建

打开"VS 开发者命令提示符"：

```bash
msbuild AutoChessDemo.slnx /p:Configuration=Debug /p:Platform=x64
```

Release 构建：

```bash
msbuild AutoChessDemo.slnx /p:Configuration=Release /p:Platform=x64
```

### 打包发布

Release 构建后，运行 `windeployqt` 收集 Qt 依赖：

```bash
cd x64\Release
windeployqt SYNERA.exe
```

将整个 `x64\Release` 目录打包为 ZIP，即可分发。

## 游戏玩法

### 界面介绍

```
┌──────────────────────────────────────────────┐
│  棋盘区域（8×8 网格）                         │
│  ┌─────────────────────────────────────┐      │
│  │ 上半区（敌方半场 — 红色）           │      │
│  ├─────────────────────────────────────┤      │
│  │ 下半区（我方半场 — 绿色）           │      │
│  └─────────────────────────────────────┘      │
├──────────────────────┬───────────────────────┤
│   备战区（8 格）      │   状态信息             │
│   [ ][ ][ ][ ]...     │   金币 / 波数 / HP    │
├────────────────────────┤   羁绊激活情况        │
│   商店（5 张卡牌）     │                       │
│   [刷新商店] [开始战斗]│                       │
└────────────────────────┴───────────────────────┘
```

- **棋盘** — 8×8 网格，上半区敌方（红）、下半区我方（绿），战斗时自动寻路攻击
- **备战区** — 8 格，用于暂存未上场单位
- **商店** — 每回合刷新 5 个随机单位，点击查看详情并购买
- **状态栏** — 金币、波数、生命值、已激活羁绊

### 基本流程

游戏共 **15 波** PvE 回合，击败全部 15 波即为通关，第 15 波为 Boss 关。

**准备阶段**：商店刷出 5 个单位 → 点击购买 → 拖拽上棋盘布阵 → 点击「开始战斗」

**战斗阶段**：单位自动寻路、攻击、施法。胜利得金币，失败扣血（HP=0 则游戏结束）。

### 操作指南

| 操作       | 方式                                                       |
| ---------- | ---------------------------------------------------------- |
| 购买单位   | 点击商店卡片预览，点击「确认购买」消耗金币入手             |
| 部署上场   | 从**备战区**拖拽到棋盘我方半场（绿色区域）                 |
| 撤回备战区 | 从**棋盘**拖拽我方单位回备战区                             |
| 出售单位   | 点击单位后在右侧面板点击「出售」，或右键备战区单位快速出售 |
| 刷新商店   | 点击「刷新商店」（第 1 次免费，之后每次递增 1 金币）       |
| 开始战斗   | 部署完成后点击「开始战斗」                                 |

### 单位介绍

| 费用 | 单位       | 生命 | 攻击 | 射程 | 羁绊       | 定位       |
| ---- | ---------- | ---- | ---- | ---- | ---------- | ---------- |
| 1★   | **步兵**   | 300  | 25   | 1    | 人类、战士 | 前排肉盾   |
| 1★   | **弓箭手** | 200  | 30   | 3    | 精灵、远程 | 后排输出   |
| 1★   | **侍从**   | 250  | 20   | 1    | 人类、侍从 | 辅助前排   |
| 2★   | **法师**   | 250  | 40   | 2    | 人类、法师 | 法术输出   |
| 2★   | **骑士**   | 500  | 20   | 1    | 人类、骑士 | 重型坦克   |
| 2★   | **治疗师** | 300  | 15   | 2    | 精灵、治疗 | 续航辅助   |
| 3★   | **刺客**   | 200  | 50   | 1    | 精灵、刺客 | 高爆发近战 |
| 3★   | **狂战士** | 400  | 45   | 1    | 兽人、战士 | 主力输出   |
| 3★   | **狙击手** | 250  | 55   | 4    | 精灵、远程 | 超远程炮台 |

单位每回合自动回满血，法力攒满自动施法。**星级倍率**：1★=1.0x → 2★=**1.8x** → 3★=**3.6x**（2★略低于 3 个 1★之和，但省人口）。

### 经济系统

- 胜利金币：前 10 轮 **波数 × 2**，第 11 轮起 **波数 × 2 + 5**
- 失败补偿：**当前波数 × 2 + 5**
- 失败扣血：**当前波数 × 4**
- HP 恢复：胜利且 HP≠100 时，恢复 **当前波数 × 2**
- 出售返还：费用 × 星级倍率（1★×1、2★×3、3★×9）

### 合星系统

- **3 个同名 1★ → 1 个 2★**（属性 ×1.8，略低于 3×1★）
- **3 个同名 2★ → 1 个 3★**（属性 ×3.6，单攻仍低于 3×2★但实战质变）
- 备战区满时购买同名同星单位会自动合星腾出空间

### 羁绊系统

场上同羁绊数量 ≥2 激活一档、≥4 激活二档，**战斗前自动为所有己方单位施加属性增益**：

| 羁绊 | 一档效果     | 二档效果       | 受益单位                     |
| ---- | ------------ | -------------- | ---------------------------- |
| 人类 | ATK+10%      | ATK+25%        | 步兵、侍从、法师、骑士       |
| 战士 | HP+10%       | HP+25%         | 步兵、狂战士                 |
| 兽人 | ATK+12%      | ATK+25%        | 狂战士                       |
| 精灵 | ATK+8%       | ATK+20%        | 弓箭手、治疗师、刺客、狙击手 |
| 远程 | 射程+1       | 射程+2         | 弓箭手、狙击手               |
| 法师 | ATK+12%      | ATK+25%        | 法师                         |
| 骑士 | HP+12%       | HP+25%         | 骑士                         |
| 治疗 | ATK+8%,HP+8% | ATK+15%,HP+15% | 治疗师                       |
| 刺客 | ATK+15%      | ATK+30%        | 刺客                         |
| 侍从 | HP+10%       | HP+20%         | 侍从                         |

### 装备系统

每 4-6 回合胜利可掉落装备，提升单位属性（ATK+15、HP+150 等），低级可合成高级。

### 棋盘席位

前 3 波最多部署 **8 人**。第 4 波起可购买额外席位（右上角按钮）：第 4-9 波 **5 金币/席位**，第 10 波起 **10 金币/席位**，永久提升部署上限。

### 难度曲线

敌方属性随波次增长：基础每波 +8%，叠加阶段系数——前期 +10%、中期 +20%、后期 +30%，后期单局战斗最长约 20 秒。

## 技术栈

| 层面     | 技术                                                     |
| -------- | -------------------------------------------------------- |
| 语言     | C++17                                                    |
| UI 框架  | Qt 6.8 (Widgets + QPainter)                              |
| 构建工具 | MSBuild / Visual Studio 2022                             |
| 音效     | Windows PlaySound API (winmm)                            |
| 寻路     | BFS 网格寻路                                             |
| 战斗     | 状态机 FSM（Idle → Moving → Attacking → Casting → Dead） |

### 核心设计

- **纯逻辑与 UI 分离**：`GameCore/` 下所有代码不依赖 Qt，数据结构与算法可独立复用，通过信号/槽与 UI 层通信
- **自绘界面**：所有 UI 组件（棋盘、备战区、商店、单位详情）使用 QPainter 自定义绘制，不依赖 QML 或 Qt Designer 控件
- **帧驱动游戏循环**：10ms QTimer（~100fps）驱动 `GameManager::Update()`，战斗逻辑固定 60fps 步进
- **分阶段生命周期**：`Preparation`（布阵/购买/合星）→ `Combat`（自动战斗）→ `CombatEnd`（结算/扣血/发金币），每个阶段有严格的操作权限控制
- **子系统解耦**：ShopSystem / SynergySystem / MergeSystem / CombatSystem / AIController 等各子系统独立，由 GameManager 协调编排
- **集中配置**：`GameConfig.h` 统一管理所有平衡性常量（棋盘尺寸、经济公式、概率表、星级倍率等），方便数值调整
